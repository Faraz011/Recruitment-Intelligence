# JD Structura 🔍

**Turn messy recruitment data into clean, queryable intelligence — instantly**

A full-stack Recruitment Intelligence Pipeline that extracts structured data from raw job descriptions and interview feedback using the Google Gemini 2.0 Flash API.

## Architecture

```
  ┌──────────────────┐
  │ Python Scraper   │
  └────────┬─────────┘
           │ sample_jds.csv
           ▼
  ┌─────────────────────┐
  │  Next.js Frontend   │◄──────┐
  │  (Dashboard & UI)   │       │
  └────────┬────────────┘       │
           │ HTTP REST           │
           ▼                     │
  ┌─────────────────────┐       │
  │  FastAPI Backend    │───────┤
  │ (Extraction Logic)  │ reads │
  └────────┬────────────┘       │
           │ API Calls           │
           ▼                     │
  ┌─────────────────────┐       │
  │   Gemini 2.0 API    │       │
  │  (Structured Data)  │       │
  └────────┬────────────┘       │
           │                     │
           ▼                     │
  ┌─────────────────────┐       │
  │   Supabase (Postgres)───────┘
  │  (Database Storage) │
  └─────────────────────┘
```

## Features

✨ **Extract Job Descriptions**

- Single JD extraction with validation
- Batch CSV processing (up to 25+ at once)
- Automatic deduplication by source URL

✨ **Extract Interview Scorecards**

- Automated scoring (Technical, Communication, Culture Fit)
- Hire recommendation logic
- Strength & weakness extraction

✨ **Dashboard**

- View all extracted job listings
- Query and filter by role, seniority, remote type, location
- Export data to CSV
- Delete records
- Pagination (20 per page)

✨ **Web Scraper**

- Fetch from RemoteOK API (~25 JDs)
- Fetch from Hacker News "Who's Hiring" threads
- Auto-clean HTML, normalize text
- Output ready-to-use CSV

## Database Schema

### Table: `job_listings`

```sql
CREATE TABLE job_listings (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  raw_text TEXT NOT NULL,
  role TEXT,
  company TEXT,
  location TEXT,
  seniority TEXT,  -- Junior, Mid, Senior, Lead, Executive, Not mentioned
  employment_type TEXT,  -- Full-time, Part-time, Contract, Internship, Not mentioned
  remote_type TEXT,  -- Remote, Hybrid, On-site, Not mentioned
  salary_min INTEGER,
  salary_max INTEGER,
  salary_currency TEXT,  -- USD, EUR, GBP, etc.
  skills_required TEXT[],  -- Array of skills
  experience_years_min INTEGER,
  experience_years_max INTEGER,
  extracted_at TIMESTAMPTZ DEFAULT now(),
  source_url TEXT,
  created_at TIMESTAMPTZ DEFAULT now()
);
```

### Table: `interview_scorecards`

```sql
CREATE TABLE interview_scorecards (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  raw_text TEXT NOT NULL,
  candidate_name TEXT,
  role TEXT,
  interviewer TEXT,
  overall_rating INTEGER,  -- 1-5
  hire_recommendation TEXT,  -- Strong Yes, Yes, Maybe, No, Strong No
  strengths TEXT[],  -- Array of strengths
  weaknesses TEXT[],  -- Array of weaknesses
  technical_score INTEGER,  -- 1-5
  communication_score INTEGER,  -- 1-5
  culture_fit_score INTEGER,  -- 1-5
  summary TEXT,  -- 2-3 sentences
  extracted_at TIMESTAMPTZ DEFAULT now(),
  created_at TIMESTAMPTZ DEFAULT now()
);
```

## Setup Instructions

### Prerequisites

- **Node.js 18+** (frontend)
- **Python 3.11+** (backend)
- **Supabase Account** (free tier works)
- **Google Gemini API Key** (free tier available)

### Step 1: Clone & Install Dependencies

```bash
# Frontend
cd frontend
npm install

# Backend
cd backend
pip install -r requirements.txt

# Scraper (optional)
cd scripts
pip install -r requirements.txt
```

### Step 2: Environment Setup

**`backend/.env`**

```env
GEMINI_API_KEY=your_gemini_api_key
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_SERVICE_KEY=your_service_key
```

**`frontend/.env.local`**

```env
NEXT_PUBLIC_API_URL=http://localhost:8000
```

Get your credentials:

- **Gemini API Key**: https://makersuite.google.com/app/apikey
- **Supabase Credentials**: Project Settings → API

### Step 3: Create Database Tables

Log into Supabase Console, go to SQL Editor, and paste:

```sql
-- Create job_listings table
CREATE TABLE IF NOT EXISTS job_listings (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  raw_text TEXT NOT NULL,
  role TEXT,
  company TEXT,
  location TEXT,
  seniority TEXT,
  employment_type TEXT,
  remote_type TEXT,
  salary_min INTEGER,
  salary_max INTEGER,
  salary_currency TEXT,
  skills_required TEXT[] DEFAULT '{}',
  experience_years_min INTEGER,
  experience_years_max INTEGER,
  extracted_at TIMESTAMPTZ DEFAULT now(),
  source_url TEXT,
  created_at TIMESTAMPTZ DEFAULT now()
);

-- Create interview_scorecards table
CREATE TABLE IF NOT EXISTS interview_scorecards (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  raw_text TEXT NOT NULL,
  candidate_name TEXT,
  role TEXT,
  interviewer TEXT,
  overall_rating INTEGER,
  hire_recommendation TEXT,
  strengths TEXT[] DEFAULT '{}',
  weaknesses TEXT[] DEFAULT '{}',
  technical_score INTEGER,
  communication_score INTEGER,
  culture_fit_score INTEGER,
  summary TEXT,
  extracted_at TIMESTAMPTZ DEFAULT now(),
  created_at TIMESTAMPTZ DEFAULT now()
);

-- Create indexes for faster queries
CREATE INDEX idx_job_listings_role ON job_listings(role);
CREATE INDEX idx_job_listings_company ON job_listings(company);
CREATE INDEX idx_job_listings_seniority ON job_listings(seniority);
CREATE INDEX idx_job_listings_remote_type ON job_listings(remote_type);
CREATE INDEX idx_job_listings_extracted_at ON job_listings(extracted_at DESC);

CREATE INDEX idx_scorecards_candidate ON interview_scorecards(candidate_name);
CREATE INDEX idx_scorecards_hire_rec ON interview_scorecards(hire_recommendation);
CREATE INDEX idx_scorecards_extracted_at ON interview_scorecards(extracted_at DESC);
```

### Step 4: Run the Scraper (Optional)

```bash
cd scripts
python scrape_jds.py
```

This generates `public/sample_jds.csv` (25+ job descriptions) that you can use in the batch upload.

### Step 5: Start Development Servers

**Terminal 1 — Backend (port 8000)**

```bash
cd backend
uvicorn main:app --reload
```

**Terminal 2 — Frontend (port 3000)**

```bash
cd frontend
npm run dev
```

Open http://localhost:3000 and you're ready to go!

## API Routes Reference

### JD Extraction

- **POST** `/api/extract` - Extract single JD

  ```json
  { "raw_text": "...", "source_url": "https://..." }
  ```

- **POST** `/api/batch` - Batch upload CSV
  - Form data: `file` (multipart/form-data)

- **GET** `/api/listings` - Get paginated listings
  - Query: `role`, `seniority`, `remote_type`, `location`, `page`

- **DELETE** `/api/listings/{id}` - Delete listing

### Scorecard Extraction

- **POST** `/api/scorecard` - Extract single scorecard

  ```json
  { "raw_text": "...", "candidate_name": "...", "role": "..." }
  ```

- **GET** `/api/scorecards` - Get paginated scorecards
  - Query: `candidate_name`, `hire_recommendation`, `role`, `page`

- **DELETE** `/api/scorecards/{id}` - Delete scorecard

### Health

- **GET** `/` - Health check → `{ "status": "ok" }`

## Usage Guide

### 📋 Single JD Extraction

1. Go to **Home** page
2. Click **"Single JD"** tab
3. Paste raw job description
4. (Optional) Enter source URL
5. Click **"Extract JD"**
6. View structured data card

### 📦 Batch Upload

1. Go to **Home** page
2. Click **"Batch Upload CSV"** tab
3. Download sample CSV template
4. Fill with job descriptions (raw_text, source_url columns)
5. Upload CSV
6. Monitor progress bar

1. Go to **Home** page
2. Click **"Interview Scorecard"** tab

1. Go to **Dashboard** page
2. View listings or scorecards
3. Use **FilterBar** to search & filter
4. Click **"Export to CSV"** to download
5. Use **Delete** to remove outdated entries

## Gemini Extraction Prompts

The backend uses two system prompts:

**JD Extraction Prompt** → Returns: `role, company, location, seniority, employment_type, remote_type, salary_min, salary_max, salary_currency, skills_required[], experience_years_min, experience_years_max`

**Interview Scorecard Prompt** → Returns: `candidate_name, role, interviewer, overall_rating, hire_recommendation, strengths[], weaknesses[], technical_score, communication_score, culture_fit_score, summary`

These are hardcoded in `backend/gemini.py` for consistency.

## Troubleshooting

**"Failed to extract JD"**

- Check API key is valid
- Ensure raw_text is not empty
- Try a longer, more detailed JD

**CSV upload fails**

- Confirm CSV has `raw_text` column
- Check file is valid UTF-8
- Try smaller batches if processing fails

**Dashboard shows no data**

- Verify database tables exist
- Check Supabase connection string
- Ensure backend is running

**Gemini returns errors**

- Verify `GEMINI_API_KEY` is set
- Check API rate limits (https://makersuite.google.com/app/usage)
- Gemini may reject very short texts

## Performance Tips

- Batch processing is more efficient than single extractions
- Use CSV import for 10+ JDs at once
- Dashboard filters are optimized with indexes
- Pagination limits: 20 records per page
- API rate limits: Follow Gemini's free tier limits (~60 requests/min)
